# Развёртывание «Волга» на сервере

Приложение: **Next.js 16 (server-rendered) + Prisma + SQLite**.
Нужен **Node.js-хостинг** (не статический и не «serverless» — требуется постоянный
Node-процесс и файловая БД). Подходит: VPS, Render (Web Service), Railway,
Timeweb Cloud, любой Node-сервер.

## Шаги запуска

1. Загрузите эту папку на сервер (или подключите как Git-репозиторий).
2. Node.js 20+ . Установите зависимости:
   ```
   npm install
   ```
   (автоматически сгенерируется Prisma-клиент под ОС сервера — шаг `postinstall`)
3. Сборка:
   ```
   npm run build
   ```
4. Запуск:
   ```
   npm start
   ```
   Сервер слушает порт из переменной окружения `PORT` (по умолчанию 3000).

Демо-данные уже лежат в `prisma/dev.db`.
Чтобы пересоздать чистую демо-базу:
```
npm run db:reset
```

## Переменные окружения
Файл `.env` включён: `DATABASE_URL="file:./dev.db"` — локальная SQLite-база, секретов нет.
Для прод-масштаба вместо SQLite подключается PostgreSQL: поменяйте `provider`/URL в
`prisma/schema.prisma` и `.env`, затем `npm run db:push && npm run db:seed`.
